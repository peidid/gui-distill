GUI AGENT DISTILLATION — RESULTS SUMMARY (learning run: Tracks A / B / C)
Updated 2026-06-02
================================================================================

GOAL
  Black-box hard-label distillation of a mobile GUI agent: train a small student
  to imitate a teacher's TEXT actions (no logits), then measure WHERE the small
  model falls short — grounding (perception) vs multi-step (control) — and how
  that depends on the TEACHER.

DESIGN — vary ONLY the teacher (label source); student + recipe + states + eval
  fixed, so any difference is attributable to the teacher.
  Student : Qwen2.5-VL-3B-Instruct, LoRA SFT (rank 16, alpha 32, lr 1e-4,
            2 epochs, bf16, vision frozen; ~30M trainable = 0.79%).
  Tracks  : A teacher = AndroidControl human demos (~error-free).
            B teacher = UI-TARS-7B-DPO   (open agent; emits 0-1000).
            C teacher = GTA1-32B         (open grounding specialist; emits PIXELS,
                                          normalized to 0-1000 for labels).
  Data    : train 800 episodes / 5167 steps; test 200 DISJOINT episodes /
            1332 steps (--skip_episodes 800; overlap=0). AndroidControl.
  Eval    : ScreenSpot-V2 grounding (400; click-in-box) + AndroidControl-test
            step accuracy (teacher-forced; type match AND args).
  COORDS  : Qwen2.5-VL & GTA1 emit ABSOLUTE PIXELS; UI-TARS emits 0-1000. All
            three STUDENTS kept the base's pixel convention (scored pixel; norm
            reads ~0). Verified from raw outputs; teacher coords handled per model.

FINDINGS (confirmed across three teachers)
================================================================================
1) MULTI-STEP TRACKS TEACHER QUALITY — monotonically.
   Order teachers by their own AndroidControl skill and the students line up:
     teacher:  human(~perfect) > GTA1 0.514 > UI-TARS 0.316 > base
     student:  A 0.552         > C 0.465    > B 0.319       > 0.114
   Stronger teacher -> stronger student on multi-step; each student lands just
   below its teacher (C 0.465<-GTA1 0.514; B 0.319<-UI-TARS 0.316).

2) GROUNDING DOES NOT TRANSFER — a hard ceiling ~0.28-0.30.
   Teacher grounding 0.650 (UI-TARS) -> 0.895 (GTA1), yet every student stays
   ~0.28-0.30 (A 0.280, B 0.280, C 0.302), far below the base (0.603) it started
   from and miles below GTA1's 0.895. Grounding is recipe/capacity-bound, NOT
   teacher-bound. (Nuance: in-distribution click improves over base — 0.125 ->
   0.46-0.48 — so it's OUT-OF-DISTRIBUTION grounding (desktop/web) that's lost.)

3) ACTION-COVERAGE / ERROR INHERITANCE, per action type.
   Students inherit what their teacher demonstrates:
     done    : A 0.815 / C 0.430 / B 0.000   (teachers: GTA1 0.275, UITARS 0.005)
     scroll  : C 0.568 > B 0.338              (teachers: GTA1 0.615, UITARS 0.243)
     type    : C 0.667 > B 0.593              (teacher GTA1 0.679)
     open_app: A 0.827 but B=C=0.133          (NO model teacher emits it: GTA1
               0.080, UITARS 0.000 -> only the HUMAN teacher taught open_app)

INTERPRETATION
  Black-box hard-label distillation scales multi-step/control behavior with
  teacher quality, but (a) grounding does not transfer (stuck ~0.30 regardless of
  teacher), and (b) students inherit the teacher's action-coverage gaps. The cost
  of the simple recipe concentrates in PERCEPTION (general grounding) and in
  capabilities NO teacher demonstrates (open_app).

CAVEATS
  - Offline teacher-forced step accuracy measures agreement with the single human
    reference action; it favors the human-distilled student and undercounts a
    capable-but-different agent. (UI-TARS 0.316 / GTA1 0.514 partly reflect
    action-space differences, not pure skill — both ground better than they
    "step-match".)
  - Single run, no seeds/error bars (pilot). ScreenSpot is OOD vs phone training.
    Teachers are OPEN (learning run); the paper's premise needs a CLOSED API.
